> **风格**：实操为主，语速适中，少废话多演示

## 第二部分：服务端安装（2:30 - 4:30）

**【旁白】**
> 假设你有一台公网 VPS，要把它做成 EasyTier 的中继节点。
> 选 **2**，进入服务端安装模式。

**【画面】** 选择 2 → 版本选择界面

```
  请选择要安装的 EasyTier 版本:
    1) v2.4.5（稳定版，默认）
    2) v2.6.0（最新版）
    3) 手动输入版本号
```

**【旁白】**
> 首先选版本。内置了稳定版和最新版，也可以手动填。这里选默认的 v2.4.5。

**【画面】** 下载方式选择

```
  请选择下载方式:
    1) 本地镜像服务器（推荐）
    2) GitHub 代理下载
    3) 直接从 GitHub 下载
```

**【旁白】**
> 然后是下载方式。国内机器建议用本地镜像或者 GitHub 代理，速度快。直连也留着给能访问 GitHub 的机器用。

**【画面】** 输入 hostname + 侦听端口

```
  ── 配置服务端信息 ──
  请输入节点主机名 (默认: my-vps): relay-01

  ── 配置侦听端口 ──
  TCP  侦听端口 [默认: 11010]:
  UDP  侦听端口 [默认: 11010]:
  WS   侦听端口 [默认: 11011]:
  WSS  侦听端口 [默认: 11012]:
```

**【旁白】**
> 填个主机名用于在拓扑里识别这台设备。然后是四个协议的侦听端口，都有默认值，直接回车就行。TCP 和 UDP 建议保持相同。
>
> 注意这些端口要在防火墙或安全组里放行。

**【画面】** 安装确认 + 自动执行

```
  ───────── 安装确认 ────────
    模式:     服务端/中继模式
    版本:     v2.4.5
    下载方式: 本地镜像
    主机名:   relay-01
    TCP 端口: 11010
    UDP 端口: 11010
    WS  端口: 11011
    WSS 端口: 11012
  确认安装？[Y/n]: Y
```

**【旁白】**
> 确认信息没问题就回车。脚本会自动完成：下载程序、解压、写入 systemd 服务、启动、显示日志。
>
> 看到"操作成功，服务运行正常！"就说明中继节点已经跑起来了。

## 第三部分：客户端安装 — 配置文件模式（4:30 - 7:00）

**【旁白】**
> 服务端好了，接下来装客户端。回到主菜单选 **1**。
>
> 客户端有两种模式：
> - **Web 控制台模式**：连接公共控制台，适合新手，简单
> - **配置文件模式**：自己指定网络名称和密钥，更灵活，功能更多
>
> 我重点演示配置文件模式，因为这个模式下可以做的事情最多。

**【画面】** 选 1 → 选 2（配置文件模式）

**【旁白】**
> 进入后会依次问你几件事：

**【画面】** 逐项填写（快放，旁白配合）

```
  ── hostname ──
  请输入机器名 (默认: nas-server): nas-home

  请输入网络名称 (network_name): my-network

  请输入网络密钥/密码 (network_secret): ********

  ── IP 地址分配方式 ──
    1) 使用 DHCP 自动分配（推荐）  ← 选这个
    2) 手动指定固定 IP

  ── 配置节点服务器地址 (peer URI) ──
    节点1已预填：tcp://et.sbgov.cn:11010
    节点2 (输入第二个节点地址...): tcp://你的VPS_IP:11010

  ── 配置子网代理地址 (proxy_network) ──
    子网 1 (输入 CIDR 如: 192.168.1.0/24): 192.168.1.0/24
    子网 2 (或直接回车完成):          ← 回车结束

  是否启用加密？[Y/n]: Y

  ── 配置侦听端口（可选） ──
    TCP  侦听端口 (直接回车=不设置):    ← 回车跳过
    UDP  侦听端口 (直接回车=不设置):
    ...
```

**【旁白】**
> 来解释一下每项：
>
> - **hostname**：就是这台设备在网络里的名字
> - **network_name / network_secret**：相当于"网络 ID 和密码"，只有同名同密的设备才能组到一起。**这很重要，别泄露。**
> - **IP 分配**：推荐 DHCP，让服务端自动分配。也可以手动指定固定 IP。
> - **peer URI**：就是你要连接的服务器地址。支持填多个，脚本会自动做冗余。第一个已经预填了公共节点，你可以再填自己的 VPS。
> - **子网代理**：这是 EasyTier 的杀手级功能。填上你本地网段的 CIDR，比如 `192.168.1.0/24`，那网络里其他设备就能直接访问你家里这个网段的所有设备了。**而且支持填多个 CIDR**，回车就结束输入。
> - **加密**：建议开启，安全性更好。
> - **侦听端口**：这是可选的。如果你想让其他节点通过这台设备中继转发，就填端口。设了的话脚本会自动加上 `relay_network_whitelist` 和 `relay_all_peer_rpc`。一般客户端不需要，直接回车跳过。

**【画面】** 安装成功 + 日志展示

**【旁白】**
> 配置会自动写入 `/etc/easytier/easytier.yaml`，同时注册 systemd 服务并启动。
>
> 全程不需要手动编辑任何文件，也不需要记一行命令。

## 第四部分：日常运维 — 修改与切换（7:00 - 9:30）

**【旁白】**
> 装好之后不是就完事了，日常维护也很重要。来看 **选项 3 修改配置**。

**【画面】** 选 3 → 配置文件模式的修改子菜单

```
  当前为客户端模式（配置文件），可执行以下操作：
    1) 修改组网信息（主机名 / 网络名称 / 网络密钥）
    2) 修改全部配置
    3) 切换为Web控制台模式
    4) 切换为服务器/中继模式
    0) 取消，返回主菜单
```

**【旁白】**
> 这里有几个亮点：
>
> **第一，修改组网信息。** 如果只想改个主机名或者换了网络密钥，选 1 就行。其他配置完全不动。
>
> **第二，修改全部配置。** 选 2 可以重新走一遍完整流程，包括改 IP 方式、换 peer 节点、加减子网代理、设置侦听端口等。
>
> **第三，也是最实用的——模式切换。**

**【画面】** 演示切换流程（快速）

```
  # 从配置文件模式切到 Web 控制台模式
  选 3 → 填用户名/机器名/控制台地址 → 确认

  # 从客户端切到服务端/中继模式  
  选 4 → 填主机名/端口 → 确认
```

**【旁白】**
> 你可以把客户端随时切换成服务端中继，或者反过来。
> **切换的时候配置文件不会被删除**，保留着方便以后切回来。
>
> 同样地，Web 控制台模式下面也有切换选项：

**【画面】** Web 控制台模式的修改菜单

```
  当前为Web控制台模式，可执行以下操作：
    1) 修改控制台信息
    2) 切换为配置文件模式      ← 一键切换
    3) 切换为服务器/中继模式   ← 一键切换
    0) 取消
```

**【旁白】**
> 三种模式之间可以任意互相切换，非常自由。

### 录制 Tips
1. **终端配色**：用深色背景 + 高对比度文字，绿/黄/红色区分明显
2. **鼠标不要乱晃**：需要指向的地方再出现鼠标，其他时候隐藏
3. **打字节奏**：真实输入，不要太快让观众跟不上
4. **错误预案**：万一演示出错（比如网络问题），自然地说一句"有时候网络会波动"，然后重试，反而显得真实
5. **BGM**：轻科技感的背景音乐，音量控制在人声的 15~20%
